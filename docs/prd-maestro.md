# PRD Maestro

## Nombre
Data-X

## Visión

Construir una plataforma documental-analítica verificable que transforme archivos reales de negocio en decisiones respaldadas por evidencia.

## Estado actual

Data-X ya tiene:
- MVP funcional
- pipeline de análisis
- frontend usable
- ingestión documental parcial
- capa de interpretabilidad básica

## Problema

Las personas trabajan con:
- PDFs
- reportes
- Excel
- CSV
- documentos mixtos

Pero la mayoría de herramientas:
- no preservan estructura
- no responden con evidencia
- fuerzan procesos manuales
- convierten el documento en texto o tabla plana demasiado rápido

## Oportunidad

Combinar:
- estructura documental
- análisis determinístico
- explicación LLM
- trazabilidad verificable

## Propuesta de valor

Subís un archivo y Data-X:
- entiende su estructura
- detecta hallazgos
- responde preguntas
- muestra de dónde sale cada respuesta

## Usuarios

- emprendedores
- pymes
- managers
- consultores
- equipos que reciben reportes y documentos difíciles de explotar

## Capacidades núcleo

- ingestión multiformato
- estructura documental persistida
- findings interpretables
- query documental
- evidencia/provenance
- historial de sesiones
- export y compartibilidad futura

## No objetivos inmediatos

- BI generalista
- dashboards arbitrarios
- enterprise complejo antes de tiempo
- features que no refuercen el plano documental

## Métrica norte

Análisis completados con al menos una interacción útil y evidencia visible.
